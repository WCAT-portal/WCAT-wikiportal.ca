Scenario A â€“ School Infrastructure Demo

A small Information Technology school is being built nearby. To facilitate learning and work, several IT services are needed. The school will require:

  • Reliable internet access for all users
  • A directory service for managing user accounts and authentication
  • Role-based user accounts
  • Internal email communications
  • A public-facing website
  • Network and system redundancy
  • Basic and advanced security protocols

The infrastructure will be segmented into a minimum of four subnets and user groups:

  • Students / Guest Access
  • Instructors
  • Staff (non-instructional)
  • IT Department

The school expects to accommodate between 250 and 500 students at any given time. Additionally:

  • There are 7 lab rooms with 25 workstations per room
  • 3 general-purpose rooms will require basic internet access

This scenario outlines the demo our group would present if contracted to build the full production environment. The mock-up includes high-level design recommendations, example configurations, and an overview of system expectations. Final construction would depend on thorough documentation provided by the school.
